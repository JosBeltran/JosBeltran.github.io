A Metadata-Driven Design

The most powerful part of the solution is that the logic is modeled as metadata rather than buried inside Apex condition blocks. Code provides the framework; metadata defines the business behavior.

  • ✓
    Highly Configurable Rules can evolve without rewriting every branch of logic.
  • ✓
    Sequence-Driven Execution Prioritize high-confidence exact matches over looser contextual ones.
  • ✓
    Explainable Logic Business analysts can read the rule hierarchy like a truth table.

Evaluating Truth with Structure

Condition Items are the atomic units of evaluation. You are no longer limited to "same email = same person." The engine supports nuanced paths based on context, meaning the CRM actively interprets business intent.

👤
Same Person, Different Interest Creates a new Opportunity instead of ignoring the lead.
⚠
Same Person, Disqualified Record Triggers a reactivation workflow with updated status.
🔒
Same Person, Protected Data Limits updates to preserve a salesperson's golden record.

From Evaluation to Orchestration

Finding a match is only half the problem. Action Groups define what manipulation happens to which object. The Logic Upsert introduces intelligent handling, resolving both identity and the correct evolutionary path of the record.

Action Items provide field-level precision, allowing targeted updates (e.g., updating the last interaction date) while preserving crucial historical context (e.g., original lead source).
